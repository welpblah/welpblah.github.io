latex -> typst

Last update: 2024-10-09
Tags: typesetting
Reading time: less than a minute

The first typsetting system setup I had was writing LaTeX in TeXShop, an application created and released in the year 2000. I would use The Comprehensive TeX Archive Network (CTAN) to look up packages and find answers on the LaTeX Stack Exchange: TeX. And while I remember trying out Overleaf, an online LaTeX editor, it never stuck.

This year, I’ve migrated over to using the new and shiny Typst typesetting system, created in 2019 and released in 2023. I first used Typst as a browser application but I prefer the simplicity of using it through the Typst CLI. I installed Typst through MacPorts package mananger (the sensible alternative to Homebrew). My setup includes the Visual Studio Code text editor with two extensions: the Typst Language Server Protocol for syntax support and vscode-pdf to view PDFs right inside the text editor.

As the screenshot illustrates, in order to use custom fonts, you have to add the custom font path when compiling: 

typst compile --font-path path/to/fonts file.typ

It seems there is partial support for hot reloading. Although, if I want to see the custom fonts, I need to recompile – I wonder if that’s the case if I have the custom fonts installed on my system.
